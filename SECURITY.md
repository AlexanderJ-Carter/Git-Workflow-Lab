## 报告安全漏洞

我们非常重视安全问题。如果你发现了安全漏洞，请**不要**在公开的 Issue 中报告。

### 如何报告

请通过以下方式之一报告安全漏洞：

1. **GitHub Security Advisories**（推荐）
   - 访问 [Security Advisories](../../security/advisories)
   - 点击 "Report a vulnerability"
   - 填写漏洞详情

2. **邮件报告**
   - 发送邮件至项目维护者
   - 主题行包含 `[Security]`

### 报告内容

请在报告中包含以下信息：

- 漏洞类型（XSS、SQL 注入、权限绕过等）
- 受影响的版本/分支
- 复现步骤
- 概念验证（PoC）代码（如果有）
- 潜在影响评估
- 建议的修复方案（可选）

### 响应时间

我们承诺：

- **24 小时内**：确认收到报告
- **72 小时内**：初步评估并回复
- **7 天内**：提供详细的评估结果
- **90 天内**：修复漏洞或发布缓解措施

### 披露政策

- 请勿公开披露未修复的漏洞
- 我们支持负责任的披露
- 修复后我们会发布安全公告
- 我们会在公告中感谢报告者（除非要求匿名）

## 安全最佳实践

使用本项目时，请注意：

### Docker 环境

- 不要在生产环境暴露 Gitea 端口
- 定期更新 Docker 镜像
- 使用强密码保护数据库

### Git 操作

- 不要在公开仓库中提交敏感信息
- 使用 `.gitignore` 排除敏感文件
- 定期检查提交历史中的敏感数据

### SSH 密钥

- 使用强密钥（推荐 ED25519）
- 不要共享私钥
- 定期轮换密钥
