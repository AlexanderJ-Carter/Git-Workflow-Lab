**适合人群**：从未使用过 Git 或刚开始学习

**学习目标**：掌握日常 Git 操作，能够独立完成代码提交和拉取

**推荐路径**：

**学习目标**：掌握分支管理、PR 工作流、冲突解决

**推荐路径**：

**适合人群**：经常遇到 Git 问题需要解决

**学习目标**：能够处理误操作、恢复丢失代码、解决各种问题

**推荐路径**：

```

**学习目标**：能够编写和维护简单的 CI/CD 工作流

**推荐路径**：

```
关卡 10 ──→ 关卡 11 ──→ 关卡 12

## 🎓 学习建议

### 学习方法

1. **边看边做**：打开课程文档，同时在终端和 Gitea 界面操作
2. **完成练习**：每课的练习题务必自己完成
3. **多做实验**：在练习环境中大胆尝试各种操作
4. **记录问题**：遇到问题记录下来，在 FAQ 或社区中寻找答案

### 学习节奏

- **每日学习**：建议每次学习 1-2 个课程，不要贪多
- **间隔练习**：学完一个阶段后，隔天再练习巩固
- **项目实战**：学完后在实际项目中应用所学知识

### 环境要求

- Docker Desktop（Windows/macOS）或 Docker + Docker Compose（Linux）
- Git 命令行工具
- 代码编辑器（推荐 VS Code）
- 现代浏览器（Chrome/Firefox/Edge）
