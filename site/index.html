Git Workflow Lab

一个把 Git 课程、浏览器实验室、仓库工程化和自动发布串在一起的教学型项目。公开站点聚焦课程与仓库本身，本地环境则提供完整的 Gitea + 终端练习体验。

查看课程目录 课程总览 学习路径 GitHub 仓库

项目亮点

这个仓库不是单纯的文档集合，而是一个把课程内容、练习环境和自动化流程放在同一个项目里的完整样例。

系统课程

当前课程覆盖 Git 基础、分支协作、历史修复、标签发布、CI/CD、自动发布等内容，并且按阶段组织学习路径。

浏览器实验室

本地模式下可组合教程、Gitea 和 Web 终端，适合教学、培训和自学演示。

真实自动化

仓库内置课程检查、GitHub Pages、Release Please 和 GHCR 发布流程，适合拿来直接参考和改造。

课程展开方式

课程不是按命令列表堆起来的，而是围绕真实协作与发布场景来设计。

阶段 0-1

基础与同步

从安装、配置、终端基础，到 clone、commit、push、pull，把 Git 日常闭环打通。

阶段 2

分支与协作

覆盖分支命名、PR、冲突处理、rebase、SSH 与团队协作约定。

阶段 3

救火与恢复

围绕 cherry-pick、revert、reflog、stash 等“出问题后怎么办”的场景设计。

阶段 4-5

发布与自动化

从 Tag、Release 到多阶段流水线、Secrets、Release Please 自动发布，逐步形成完整工程化链路。

仓库里有什么

docs/

课程正文、总览、学习路径和本地文档查看器。

site/

对外展示首页和本地工作台入口。公开模式与本地模式会显示不同重点。

.github/workflows/

课程检查、Pages、Release Please、容器构建与发布都在这里。

docker-compose.yml

定义本地浏览器实验环境，是把课程变成可操作平台的关键入口。

如果你是在 GitHub Pages 上查看此页面，重点入口是课程目录、课程总览和学习路径；如果你在本地打开，则会额外看到实验环境入口。
